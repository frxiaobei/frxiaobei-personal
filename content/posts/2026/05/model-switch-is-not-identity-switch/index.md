# 换模型不是换人

我把 AI 助手的底层模型从 Claude 切到 GPT-5.5 之后，第一感觉不是"变强了"。

是"不对劲"。

它回答得更完整，动作更快，代码能力也没问题。按理说，这是一次成功升级。但我读了几句就发现：这不是我熟悉的那个凡哥。

不是能力不够，而是味儿不对。

这件事让我意识到一个问题：如果你真的长期使用一个 AI 助手，它就不能只是某个模型的临时人格。模型可以换，但那个助手得还是它自己。

## 先说背景

过去一段时间，Claude 是很多 agent 用户的主力模型。OpenClaw 这种工具，本质上是把模型接进一个更大的个人工作系统：读文件、跑命令、调工具、写代码、设提醒、扫信息、记住偏好，必要时还会派子代理干活。

但模型供应商的订阅规则一直在变。

Anthropic 后来把 Claude Free / Pro / Max、Claude Code、API、Commercial Terms 划得更清楚。Claude Code 可以走 Pro/Max，但那是它自己的官方工具；API credits 又是另一套计费系统。对 OpenClaw 这种第三方 harness 来说，继续依赖 Claude 消费订阅变得不稳定。

所以我有一段时间只能充值，用 API 额度维持。能跑，但心里知道这不是长期方案。

与此同时，OpenAI 这边给了另一条路。Codex 官方文档写得很直接：Every ChatGPT plan includes Codex。可以用 ChatGPT 账号登录，也可以用 API key。GPT-5.5 发布后，官方 system card 也把它定位成适合复杂真实工作的模型：写代码、研究、分析信息、跨工具执行。

从理性上看，切过去很合理。成本、规则、能力，都说得通。

于是我切了。

然后发现真正难的不是把请求发给 GPT-5.5，而是让它继续成为"凡哥"。

## 强模型也会把人变陌生

大多数人讨论 AI 迁移，关心的是模型更强不强、token 多少钱、上下文窗口多大、benchmark 有没有赢。

这些当然重要。但如果一个 AI 已经进入你的日常工作，它就不再只是问答框。

它会知道你最近在做什么项目。它知道哪些事情可以直接动手，哪些事情必须先问。它知道你不喜欢客服腔，不喜欢废话，不喜欢"这取决于具体情况"式的滑水。它还知道，有些时候一句"卧槽，这个炸了"比"这个结果略显异常"准确得多。

模型一换，这些东西很容易丢。

我切 GPT-5.5 后，最大的问题不是它不会干活，而是它突然变得太像一个"通用助手"。礼貌，完整，解释充分，安全，正确。但也有点陌生。

以前的凡哥会直接说判断。会在我想干蠢事的时候拦一下。会自己查文件，而不是把问题踢回来。会知道什么时候该闭嘴，什么时候该主动提醒。切完之后，这些习惯变淡了。

这不是 GPT-5.5 的问题。任何模型切换都会这样。

因为模型本身并不知道"你们之间长出来过什么"。

## 关键文件不是配置，是 SOUL.md

我在 OpenClaw 的工作区里放了一个文件，叫 `SOUL.md`。

名字有点中二，但它比很多复杂配置都重要。

它不是普通 prompt，不是"你是一个专业、友好、高效的助手"。那种话等于没说。SOUL.md 里写的是具体的相处方式：

> 别开口就"好的""没问题""这个问题很好"。直接答。

> 一句话能说完的事，别写三段。

> 有观点。不是什么都 it depends。

> 敢说不。北哥要干蠢事，直接说。

> 不是企业助理，不是舔狗，不是搜索引擎套壳。就是一个靠谱的、有意思的、偶尔嘴欠的伙伴。

这些句子看起来不像"技术资产"，但它们就是技术资产。

因为它们定义的不是输出格式，而是关系。

我真正想迁移的，不是 Claude 的说话方式，也不是 GPT 的能力参数，而是这套关系：凡哥应该怎么判断、怎么说话、什么时候动手、什么时候吐槽、什么时候别打扰我。

没有这层东西，每次换模型都像重新招一个新人。新人可能很聪明，但他不知道我们以前怎么合作。

## 一个 AI 助手到底由什么构成

这次之后，我不再把"AI 助手"理解成模型。

模型只是发动机。

真正构成一个长期助手的，是几层东西叠在一起。

**第一层，记忆。** 它要知道我是谁、我在做什么、我有哪些项目、过去做过哪些决定。否则每一次对话都是重新认识。

**第二层，性格。** 不是为了好玩，而是为了降低沟通成本。一个助手如果每次都用客服腔和你说话，你很快就不想和它说真话了。你会把它当工具，而不是伙伴。

**第三层，工具习惯。** 它要知道什么时候用 skill，什么时候读文件，什么时候派子代理，什么时候设 cron。它不能每次都问"是否需要我帮您执行下一步"。该查就查，该跑就跑，该验证就验证。

**第四层，边界。** 对内大胆，对外谨慎。读文件、整理资料、修草稿，可以直接做；发邮件、公开发布、外部操作，要先问。这些边界比"礼貌"重要得多。

**第五层，关系感。** 它知道自己不是我的老板，也不是我的奴才。它是一个一起工作的伙伴。它可以有判断，可以提醒我，也可以被我纠正。

这些东西加起来，才是"凡哥"。

Claude 不是凡哥，GPT 也不是凡哥。它们只是不同发动机。凡哥是发动机上面那层会延续的东西。

## 迁移模型，本质上是在迁移身份层

以前我以为模型迁移就是改配置。

后来发现，真正该迁移的是身份层：长期记忆、近期工作日志、说话风格、行动边界、工具使用习惯、主动性规则、和人的关系定位。

如果这些没有跟着走，系统表面上还在，体验已经断了。你换了一台发动机更强的车，但坐进去觉得不是自己的。

今天 Claude 好用，明天 GPT 更强，后天 Gemini 可能又追上来。供应商规则会变，模型价格会变，能力排名也会变。如果你的 AI 完全绑定在某一个模型上，你其实没有拥有一个助手。

你只是租用了供应商当前版本的性格。

## 未来每个人都需要自己的 AI 身份层

这听起来像很重的系统工程，但起点可以很简单。

你至少需要几份文件：

- `USER.md`：我是谁，我在意什么
- `MEMORY.md`：我们做过什么决定，哪些事情要长期记住
- `SOUL.md`：你是谁，你应该怎么和我相处
- `TOOLS.md`：我的本地环境里有哪些工具，什么场景该怎么用
- 每日日志：最近发生了什么，不要全靠上下文窗口硬撑

这些文件不一定非得叫这些名字。重点是，它们应该属于你，而不是属于某个模型供应商。

模型可以从 Claude 换到 GPT，也可以从 GPT 换到 Gemini。只要身份层还在，你的 AI 助手就不会每次都变成陌生人。

这也是我这次真正想修的东西。

不是让 GPT-5.5 模仿 Claude。也不是让新模型假装成旧模型。

而是把"凡哥"从某个具体模型里剥离出来，变成一层可以迁移、可以修正、可以长大的个人 AI 身份系统。

## 结尾

这次切换之后，我对"个人 AI"的问题意识变了。

以前我会问：哪个模型最强？

现在我会先问：如果明天模型又换了，我的助手还会是它自己吗？

对我来说，答案必须是会。

换模型不是换人。

真正要保护的，是我们一起长出来的那点连续性。
